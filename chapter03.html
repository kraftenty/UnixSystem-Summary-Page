Chapter 03

Users and Groups

Unix system has 3 primary types of accounts: root accounts, system accounts, and user accounts.

  • 1. Root Accounts, also known as Super user, have complete and unfettered control of the system.
    The root account has no restrictions and can do anything on the system.
  • 2. System Accounts are needed for the operation of system-specific components.
    Examples of system accounts include mail, bin, adm, daemon, noaccess, and nobody accounts.
  • 3. User Accounts are normal accounts used by ordinary users.
    User accounts have limited access to the system.

There are 3 important files which contain user information in a Unix system: /etc/passwd, /etc/group, and /etc/shadow.

  • 1. The /etc/passwd file identifies the authorized accounts for the system.
    It contains user information like login ID, UID (user ID), GID (group ID of primary group), comments, home directory,
    and login shell of each user.
  • 2. The /etc/group file contains the secondary group of each user.
    It contains group name, GID, and members.
  • 3. The /etc/shadow file contains encrypted passwords of each user.
    It cannot be read by humans because it is encrypted.

This is some command about this chapter:

$ su: switch user (substitute user). Users can change the current user to another user by using this command. For example, $ su unix345 can change the user from the current user to the unix345 user. However, this command cannot change the working environment.

$ su - unix345 can change the user from the current user to the unix345 user, and even the working directory. It changes completely. You can check this with $ pwd command.

$ su with no argument can switch to the root user.

And you can return back to the original user by typing $ exit.
