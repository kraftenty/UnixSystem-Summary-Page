Chapter 05

Customizing Working Environment

Environment Variable

  1. Environment variable affcts the look and feel of computing experiences.
  2. $PS1 : commands prompts. We can change prompt to just one dollar by PS1='$'.
  3. when we type $ echo $PS1, there will be \u@\h:\w\$. ex) 20101192@comunix:~$
  4. \u means username.
  5. \h means hostname.
  6. \w means path.
  7. $PATH : it contains a list of directory that are searched when a user input a command.
  8. for example, echo $PATH's result is PATH=/usr/bin:/usr/ucb. This means PATH's value is 2(/usr/ucb and /usr/bin). colon is seperator.
  9. When you want to append some value behind PATH value, use PATH=$PATH:/usr/sbin.

Local/environment variable

  1. Local variable : variable that affects to current shell only.
  2. ex) a=3 : set a's value by 3. you can check it by using echo $a.
  3. you can delete this value by using $ unset a.
  4. $ export a : set a as environment variable. It affects to sub-shell too.
  5. This is Example.
  6. $ a=3; b=5 //set two variables
  7. $ echo $a $b //print a and b's value
  8. $ export b //set b as environment variable
  9. $ sh //start sub-shell
  10. $ echo $a //no result. a is not a environment variable
  11. $ echo $b //print 5
  12. $ exit //return to parent(original) shell

Variable commands

  1. $ set : see all variables and its value.
  2. $ set | grep PATH : see PATH's value
  3. $ env : see all environment variables and its value
  4. $ echo $a : print variable a's value
  5. $ export var : makes variable var a viable to an environment variable
  6. $ unset a : unset variable a

System/User variable

  1. System varialbe: Embedded variable. Use only. ex)PS1, PATH, SHELL, Lang,...
  2. User variable: User defined variable. ex)a=3

Absolute/Relative Path

  1. Absolute Path : starts with /(root) path. ex)/usr/bin/ls
  2. Relative Path : doesn't start with /. reference is current directory. ex)dir1/hi.txt
  3. Shorthand notations
  4. . : current directory. ex)./.profile
  5. .. : parent directory. ex)../dir2/hi.txt
  6. ~ : user's home directory ex)~unix212/hi.txt

Kinds of Shell

  1. Bourne shell(sh) : original unix shell.
  2. C shell(csh) : syntax is similar with C programming language.
  3. Korn shell(ksh) : sh+csh+additional functions.
  4. Bourne Again Shell(bash) : sh+many functions.

Shell initialization files
(Auto-run file when user log-in shell)

  1. $ etc/profile : global configuration file.
  2. Sh : ~/.profile
  3. Csh : ~/.cshrc, ~/.login
  4. Ksh : ~/.profile, ~/.kshrc
  5. Bash : ~/.bashrc_profile, ~/.profile
  6. when user logined, /etc/profile, ~/.profile, ~/.bashrc_profile effects user's shell.

Alias

  1. Substituting one term to another
  2. $ alias : print all alias
  3. $ alias ls='ls -l' : set ls as ls -l
  4. $ unalias ls : unset ls alias

  5. $ alias c='clear' : bash shell
  6. % alias c clear : cshell
  7. $ c() clear : sh
