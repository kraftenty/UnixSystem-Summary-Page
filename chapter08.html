Chapter 08

Advanced Tools

Regular Expression

  1. A syntactical set or phrase that represents a pattern of text or strings.
  2. It enables you to represent a varying array of characters with a much smaller set of predefined characters.
  3. It often include metacharacters
  4. Metacharacters : characters that represent another set or group of characters or commands.

Meta characters

  1. . (dot,period) : a character
  2. [] (square brackets) : any one of inside in it
  3. * (asterisk,splat) : zero or more characters
  4. [^.*] (caret) : do not match any one of inside in it
  5. ^.* (caret) : match at the beginning
  6. ^.*[.*] : match at the beginning and match inside in it
  7. ^ (caret) : match at the start of the line (start with)
  8. $ (dollar) : match at the end of line (end with)
  9. \ (backslash) : remove special meaning of the following characteer
  10. ? (question mark) : matches zero or one character
  11. [a-z] : match all occurrences of any single letter
  12. [0-9] : match all occurrences of any single digit

  13. [0-9][0-9][0-9]\.[0-9][0-9][0-9]\.[0-9][0-9][0-9] --> U.S. Telephone number

FTP(file transfer protocol)

  1. C:\> ftp 10.10.54.75
  2. id/password
  3. > cd public_html
  4. > dir (or ls)
  5. > asc //ascii mode:source code, text file, etc..
  6. > put a.html
  7. > get b.html
  8. > mput *.html
  9. > bi // binary mode:other files like jpg..
  10. > put my.jpg
  11. > quit

grep

  1. grep : print lines that has the string
  2. $ grep unix30 /etc/passwd : grep the 'unix30' in passwd file
  3. $ cat /etc/passwd | grep unix30 : grep the 'unix30' in passwd file. same with above
  4. $ grep -v unix30 /etc/passwd : print passwd file without 'unix30'. opposite resuit.
  5. $ grep -i unix30 /etc/passwd : grep the 'unix30' in passwd file case insensitive.(ignore case)
  6. $ grep '^root' /etc/group : grep starting with root in passwd file.
  7. $ grep 'adm$' /etc/group : grep end with adm in passwd file.

find

  1. Find files in directory structure.
  2. $ find / -name core : find file on root directory that name is 'core'
  3. $ find /export -user unix300 : find file on /export directory that's belonged to unix300
  4. $ find /export -size +2000k : find file on /export directory that's size is bigger than 2000 kilobytes(2MB).
  5. $ find / -perm -744 : find file on root directory that's permission is 744.
  6. $ find / -name core 2>/dev/null : find file on root directory that's name is core, and delete error message into null device

sort

  1. $ sort /etc/group : sort the group file by alphabet ascending order.
  2. $ sort -t: +1 /etc/group : sort the group file by gid number order.(seperated by colon)
  3. -f : ignore case(capital)
  4. -r : reversed order(descending order)
  5. -t : set delimiter(field seperator)
  6. -o : set output file.
  7. $ sort /etc/group -o out.txt : sort the group file and save at out.txt
  8. $ sort /etc/group > out.txt : sort the group file and save at out.txt(same as above)

Tee/script

  1. Tee : split output(screen and file). Simply, output at screen and file simultaneously.
  2. $ ps -ef | tee ps.txt : print result of ps -ef at screen, and save as ps.txt
  3. $ echo hi | tee hi.txt : print hi and save in hi.txt
  4. $ telnet 10.10.54.75 | tee telnet.txt : telnet to 10.10.54.75 and all screen output at telnet is saved in telnet.txt

  5. script : record interactive login session.
  6. $ script [-a filename] : after run this command, all screen output is saved in 'typescript' file until exit command executes.

Unix utilities

  1. wc : word count
  2. cut : extract character or words
  3. paste : file merge
  4. tr : translate character
  5. split : file divide
  6. uniq : file compare
  7. cmp : file compare
  8. comm : file compare
  9. diff : file compare
